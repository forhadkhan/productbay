# Gutenberg Blocks Architecture

ProductBay provides a native, high-performance block-based editing experience for incorporating product tables into any WordPress page or post. The block system consists of two primary blocks: **Product Table** and **Product Table (Tab)**.

---

## 🏗 High-Level Block Architecture

ProductBay blocks follow a **Dynamic Block** pattern, utilizing WordPress's `ServerSideRender` (SSR) component in the editor and dedicated PHP render callbacks for the frontend. This ensures that the frontend output is always in sync with the core table rendering logic used by shortcodes.

### Core Components

| Component | Responsibility |
|-----------|----------------|
| **[BlockManager.php](../app/Blocks/BlockManager.php)** | Orchestrates registration and asset enqueuing for all blocks. |
| **[ProductTableBlock.php](../app/Blocks/ProductTableBlock.php)** | Render handler for single tables. |
| **[TabProductTableBlock.php](../app/Blocks/TabProductTableBlock.php)** | Render handler for the complex tabbed layout. |
| **`src/blocks/`** | React-based editor implementation (Inspector controls, placeholders). |
| **`blocks/`** | Compiled block metadata (`block.json`) and build artifacts. |

## 🧩 Block Index

### 1. Product Table Block (`productbay/product-table`)
A straightforward block that allows users to select a single product table from their library.

- **Attributes**: `tableId` (int).
- **Supports**: 
    - **Color**: Background, Text, and Link colors.
    - **Spacing**: Padding and Margin.
    - **Border**: Radius, Style, and Width.
- **Onboarding**: Includes an "in-block" selection dropdown when no table is assigned.

### 2. Product Table (Tab) Block (`productbay/tab-product-table`)
An advanced block that wraps multiple product tables into an interactive, accessible tabbed interface.

- **Attributes**: `tableIds` (array), `tabLabels` (array), `activeTab` (int).
- **Supports**: Same as Product Table block (Color, Spacing, Border).
- **Interactivity**: 
    - **Editor**: Custom event interception allows switching between tabs directly in the Gutenberg canvas.
    - **Frontend**: Lightweight vanilla-JS switcher (`block-tabs.js`) handles instantaneous tab switching without page reloads.
- **Accessibility**: Implements full ARIA roles (`tablist`, `tab`, `tabpanel`) with keyboard navigation support.

---

## 🎨 Styling & Theming

ProductBay blocks utilize the modern **WordPress Style Engine**. By leveraging `get_block_wrapper_attributes()` in our PHP render callbacks, we automatically apply classes and inline styles generated by the user's sidebar settings (Margin, Padding, Colors).

### Asset Loading Strategy

To ensure optimal performance, block-specific assets are only loaded when the blocks are present on the page:

- **Frontend CSS**: `block-tabs.css` provides scoped styling for the tabbed layout.
- **Frontend JS**: `block-tabs.js` is a ~1KB vanilla script (zero dependencies) that initializes the tab switching behavior.
- **Editor Accuracy**: We use `add_editor_style()` and `enqueue_block_editor_assets` to ensure that our styles are forced into the editor's iFrame, providing an accurate 1:1 preview.

## 🛠 Development Workflow

### Adding New Attributes
1. Define the attribute in the block's `block.json`.
2. Add the corresponding control to the `edit.js` within `InspectorControls`.
3. Update the PHP `render()` method to handle the new attribute.

### Build System
The build system is powered by `@wordpress/scripts` but customized via `webpack.config.js` to support a multi-entry approach. This allows us to keep block source code in `src/blocks/` while outputting production-ready metadata to the root `blocks/` directory, adhering to WordPress registration standards.

```bash
# Compile blocks and admin assets
bun run build
```
