# Request a Feature

Have an idea for a new feature or improvement? We'd love to hear from you!

## How to Submit a Feature Request

1. Visit the [GitHub Issues page](https://github.com/forhadkhan/productbay/issues)
2. Click **"New Issue"**
3. Choose the **"Feature Request"** template (if available) or create a blank issue
4. Fill in the details:

### What to Include

- **Title**: A clear, concise title for your feature
- **Description**: Explain what you'd like to see and why
- **Use Case**: How would this feature help you or your customers?
- **Examples**: Screenshots, mockups, or links to similar features in other plugins
- **Priority**: How important is this for your workflow?

## Feature Request Template

You can copy and paste this template when creating your GitHub issue:

```markdown
## Feature Request

**Summary:**
[Brief description of the feature]

**Problem:**
[What problem does this solve?]

**Proposed Solution:**
[How do you envision this working?]

**Use Case:**
[Describe a real-world scenario where this would be useful]

**Additional Context:**
[Screenshots, mockups, links, or anything else helpful]
```

## Guidelines

- ✅ **Search first** — Check if a similar request already exists. If it does, add a 👍 reaction and comment with your use case.
- ✅ **Be specific** — The more detail you provide, the better we can evaluate and prioritize your request.
- ✅ **One feature per issue** — If you have multiple ideas, create separate issues for each.
- ❌ **Don't include support requests** — For bugs or help, use the [support forum](https://wordpress.org/support/plugin/productbay/) instead.

## What Happens Next?

1. We review all feature requests regularly
2. Popular and feasible requests get added to our development roadmap
3. We may ask follow-up questions for clarification
4. When a feature is implemented, the issue will be closed with a reference to the changelog entry

Thank you for helping make ProductBay better! 🙏
